# 슈퍼 토스 백테스팅 기능 인수인계

이 문서는 현재 앱에서 개발된 **백테스팅 기능만** 설명합니다. API Key, Secret Key, 토큰, 개인 계정 정보, `.env.local` 내용은 포함하지 않았습니다.

## 1. 기능 목적

백테스팅은 사용자가 선택한 관심주식과 매매 기준을 넣으면, 선택 날짜의 실제 가격 데이터를 우선 조회해서 전략 결과를 보여주는 기능입니다.

현재 기능은 실제 증권사 주문 API나 실거래 주문을 호출하지 않습니다. 가격 데이터만 외부 공개 시세 API에서 가져오고, 매매는 화면 안에서 **모의 계산**만 합니다.

## 2. 위치

- 주요 코드: `src/App.tsx`
- 백테스팅 화면 스타일: `src/App.css`
- 로컬 시세 프록시: `vite.config.ts`
- 차트 라이브러리: `echarts`
- 실행 방식: React + TypeScript + Vite 단일 프론트엔드 앱

## 3. 사용자가 입력하는 값

백테스팅 화면에서 아래 값을 입력하거나 선택합니다.

| 항목 | 의미 |
| --- | --- |
| 투자 날짜 | 결과 카드에 표시되는 기준 날짜 |
| 관심주식 | 저장된 종목 중 백테스트할 종목 |
| 시작가 | 매수 기준 가격 |
| 투자 원금 | 시작 보유분과 하락 매수 예비 현금을 포함한 총 운용금 |
| 수량 | 시작 보유 주식 수 |
| 상승 판매 1~3단계 | 시작가 대비 이 수익률 이상이면 단계별 일부 판매 |
| 하락 매수 1~3단계 | 시작가 대비 이 하락률 이하이면 예비 현금으로 단계별 추가 매수 |

관심주식을 바꾸면 해당 종목에 저장된 시작가, 수량, 상승 판매 1단계, 하락 매수 1~3단계 기준이 자동으로 입력됩니다.

## 4. 기본 샘플 종목

현재 기본 저장 종목은 2개입니다.

| 종목 | 시장 | 시작가 | 수량 | 상승 판매 1단계 | 하락 매수 1단계 | 하락 매수 2단계 | 하락 매수 3단계 |
| --- | --- | ---: | ---: | ---: | ---: | ---: | ---: |
| 한미반도체 `042700` | 한국장 | 189,400원 | 30주 | +5% | -4% | -8% | -12% |
| NVIDIA `NVDA` | 미국장 | $142.83 | 20주 | +4% | -3% | -7% | -10% |

## 5. 계산 방식

핵심 계산 함수는 `buildBacktestResult()`입니다.

계산 순서는 다음과 같습니다.

1. 시작 평가금 계산  
   `시작가 x 수량 + 하락 매수용 예비 현금`

2. 실제 가격 데이터 조회  
   미국장 종목은 EODHD demo 일봉을 먼저 사용합니다. 실패하면 Yahoo Finance 5분봉/일봉을 시도합니다.

3. 샘플 가격 폴백  
   외부 시세가 막히거나 해당 날짜가 휴장일이면 `intradayMultipliers` 샘플 가격 흐름으로 폴백하고, 결과 카드에 경고를 표시합니다.

4. 각 시간대마다 조건 확인  
   가격이 움직일 때마다 상승 판매 3단계와 하락 매수 3단계 조건을 순서대로 검사합니다.

5. 이벤트 발생 시 보유 수량과 현금 변경  
   판매하면 주식 수량은 줄고 현금은 늘어납니다. 하락 매수하면 주식 수량은 늘고 현금은 줄어듭니다.

6. 매 시점 평가금 계산  
   `현금 + 현재 보유 수량 x 현재 가격`

7. 최종 결과 계산  
   최종 평가금, 수익금, 수익률, 최고가, 최저가, 최고 수익률 시점, 최대 손실 시점을 계산합니다.

## 6. 매매 규칙

현재 규칙은 한 번 발생한 조건이 중복 실행되지 않도록 설계되어 있습니다.

| 조건 | 실행 내용 | 실행 수량 |
| --- | --- | --- |
| +3% 이상 | 상승판매 1단계 | 시작 보유 수량의 20% |
| +5% 이상 | 상승판매 2단계 | 시작 보유 수량의 30% |
| +8% 이상 | 상승판매 3단계 | 시작 보유 수량의 50% |
| -3% 이하 | 하락매수 1단계 | 총 운용금의 10% 이내 |
| -6% 이하 | 하락매수 2단계 | 총 운용금의 15% 이내 |
| -9% 이하 | 하락매수 3단계 | 총 운용금의 25% 이내 |

수량 계산 시 최소 1주는 거래되도록 처리합니다.

## 7. 화면에 표시되는 결과

백테스팅 실행 후 화면에는 아래 결과가 표시됩니다.

| 화면 영역 | 표시 내용 |
| --- | --- |
| 결과 카드 | 최종 수익금, 최종 수익률, 투자 가정 설명 |
| 데이터 출처 | EODHD/Yahoo/샘플 중 어떤 데이터로 계산했는지 |
| KPI 카드 | 시작 평가금, 마감 평가금, 최고가, 최저가 |
| 시뮬레이션 차트 | 장중 가격 흐름, 전략 이벤트 지점 |
| 현재 시점 패널 | 현재 시간, 가격, 보유 수량, 현금, 평가금액, 누적 수익금, 전략 상태 |
| 이벤트 타임라인 | 조건 발생 시간, 판매/매수 내용, 당시 수익률 |
| 최종 요약 | 최종 수익금, 최종 수익률, 판매/매수 횟수, 최고/최대손실 |

## 8. 시뮬레이션 기능

백테스팅 결과는 단순 표가 아니라 재생 가능한 시뮬레이션으로 볼 수 있습니다.

- `재생`: 장중 흐름을 시간 순서대로 재생합니다.
- `일시정지`: 현재 위치에서 멈춥니다.
- `다시 재생`: 끝까지 본 뒤 처음부터 다시 재생합니다.
- `처음으로`: 진행 상태를 0으로 되돌립니다.
- `1x / 2x / 5x / 10x`: 재생 속도를 바꿉니다.
- 이벤트 항목 클릭: 해당 매매 이벤트 시점으로 바로 이동합니다.
- `시뮬레이션 / 결과표` 전환: 그래프 화면과 이벤트 표를 바꿔 봅니다.

차트는 `echarts`로 그립니다. 가격 흐름은 선 그래프이고, 판매/매수 이벤트는 점으로 표시됩니다.

## 9. AI 추천값 적용 버튼

`AI 추천값 적용` 버튼은 현재 화면의 추천 종목 중 1순위 값을 백테스팅 폼에 넣는 역할입니다.

주의할 점:

- 실제 AI API 호출을 하지 않습니다.
- 현재 앱 내부 샘플 추천 데이터에서 값을 가져옵니다.
- 상승 판매, 하락 매수 3단계 기준을 자동 입력합니다.
- 적용 후 바로 계산하지 않고, 사용자가 `백테스팅 실행`을 눌러야 결과가 갱신됩니다.

## 10. 현재 한계

현재 백테스팅은 데모/프로토타입 성격입니다.

- 미국장 일봉은 EODHD demo 데이터를 우선 사용합니다.
- Yahoo Finance는 요청 제한이 걸릴 수 있어 실패 시 샘플로 폴백합니다.
- 한국장 실제 데이터는 Yahoo가 막히면 정식 API 키나 다른 시세 소스가 필요합니다.
- 수수료, 세금, 환율, 슬리피지, 체결 실패는 반영하지 않습니다.
- 매매 조건은 하루 안에서 1회씩만 실행됩니다.
- 실거래 주문과 연결되어 있지 않습니다.
- 결과는 투자 조언이 아니라 전략 화면 검증용 모의 결과입니다.

## 11. 다음 개발자가 이어서 할 일

실제 서비스 수준으로 확장하려면 아래 순서가 좋습니다.

1. 정식 시세 API 키 연결  
   EODHD, Toss Invest Open API, Alpha Vantage 등 실제 운영용 API 키를 연결하면 요청 제한과 한국장 제한을 줄일 수 있습니다.

2. 백테스팅 엔진 분리  
   현재 `src/App.tsx`에 들어있는 `buildBacktestResult()`를 별도 파일로 분리하면 테스트와 유지보수가 쉬워집니다.

3. 비용 반영  
   수수료, 세금, 환율, 슬리피지를 계산에 넣어 실제 결과와 차이를 줄입니다.

4. 기간 백테스트 추가  
   하루 단위가 아니라 여러 날짜를 선택해 누적 수익률, 최대 낙폭, 승률을 계산합니다.

5. 결과 저장  
   실행한 백테스트 조건과 결과를 저장해서 나중에 비교할 수 있게 합니다.

6. 테스트 코드 추가  
   상승 판매 3단계와 하락 매수 3단계 조건이 정확히 작동하는지 자동 테스트를 만듭니다.

## 12. 인수인계 시 주의사항

- `.env.local`은 전달하지 마세요.
- 실제 API Key, Secret Key, Access Token은 이 문서와 코드 공유본에 넣지 마세요.
- 현재 백테스팅은 API 없이도 동작합니다.
- 다른 개발자가 실행하려면 의존성 설치 후 아래 명령을 쓰면 됩니다.

```bash
npm install
npm run dev
```

빌드 확인은 아래 명령입니다.

```bash
npm run build
```
